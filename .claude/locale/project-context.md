# Project Context — Mahendra Portfolio

## Owner
Mahendra Singh Puniya — Senior Software Engineer at Primathon. Claude Premium + Anthropic API subscriber. Prefers AI-driven workflows, maximum automation. High-volume blog publishing (10-15/day target).

## Stack
- Next.js 16.2.2 + React 19 + TypeScript strict (no `any`)
- Tailwind CSS 4 + Framer Motion + Three.js + GSAP
- Sanity CMS (free plan) for blog
- Supabase for auth/admin
- Vercel for deployment
- Domain: pooniya.com

## Key Architecture Decisions
- Server Components by default; "use client" only when hooks/events needed
- Three.js lazy-loaded with next/dynamic ssr:false; mobile gets gradient fallback
- Blog automation via scripts/blog.ts (not Sanity Studio)
- FontAwesome kept (deeply integrated, 16 files) but tree-shaking optimized
- GA4 + GTM + Search Console for analytics; custom blog engagement tracking
- RSS feed at /feed.xml
- Sitemap auto-generated with dynamic Sanity blog posts

## Blog System
- Content: Sanity CMS (post + category schemas)
- Automation: scripts/blog.ts (new, publish, batch, list, delete)
- Markdown files: content/blog/*.md with YAML frontmatter
- Publishing: `npm run blog:batch` publishes all new .md files
- Tracking: content/blog/.published.json (gitignored)
- Blog writing style: Senior dev voice, emojis, tables, humor, first-person
- Related posts auto-generated by category overlap
- Lazy loading blog grid with device-specific layouts

## Analytics
- GA4: NEXT_PUBLIC_GA_MEASUREMENT_ID
- GTM: NEXT_PUBLIC_GTM_ID (optional)
- Search Console: NEXT_PUBLIC_GOOGLE_SITE_VERIFICATION
- Custom events: blog_view, blog_scroll_depth, blog_time_spent
- Blog SEO: BlogJsonLd (BlogPosting + BreadcrumbList), canonical URLs, OpenGraph

## External Services (all graceful-fallback)
- Sanity CMS — blog content
- Supabase — auth + admin
- Resend — contact form email
- Web3Forms — contact form (primary)
- Google Analytics 4 — analytics
- HCaptcha — form protection

## DO NOT
- Don't use `any` types (TypeScript strict)
- Don't add "use client" to server components unnecessarily
- Don't upload images to Sanity (free plan quota) — use external URLs
- Don't write dry/textbook blog content — always senior dev voice with personality
- Don't hardcode content in components — use src/data/ files
- Don't create components over 200 lines without splitting
- Don't use generic fonts — always Outfit/Plus Jakarta Sans/JetBrains Mono via next/font
- Don't skip metadata on new pages — every page needs title, description, canonical, OG
- Don't forget to run `npm run build` to regenerate sitemap after adding blog posts
