Three.js: 3-tier performance system in HeroScene.tsx (high/medium/low) based on devicePixelRatio, hardwareConcurrency, deviceMemory. Low tier: no post-processing, 1200 particles, DPR 1, low-power GPU. Medium: bloom + vignette only, 2500 particles. High: all effects (bloom, chromatic aberration, vignette, noise), 4000 particles, multisampling 4. Phones (<768px, via useHeavyVisuals) skip Three.js entirely and get the CSS gradient-mesh fallback (HeroBackdrop in HeroSection.tsx) — no 3D bundle download at all. FontAwesome: global library.add() removed — icons imported directly per component for tree-shaking. Blog grid: lazy loads 6 posts at a time via IntersectionObserver. Mobile gets compact card layout (PostCardCompact). prefers-reduced-motion: all infinite CSS animations disabled. gsap/@gsap/react/lenis were removed from package.json (Sept 2026 perf pass) — SmoothScrollProvider is now a pass-through; Lenis caused wheel-scroll lock-ups (see comment in SmoothScroll.tsx) and nothing else consumed it. Hero entrance animation runs on pure CSS (globals.css `.hero-fade-*` classes), not framer-motion, so it's visible at first paint instead of waiting on JS hydration. PortfolioLoader (src/components/brand/) shows only on first session visit, backed by a blocking pre-paint script + CSS `data-loader` backdrop so there's no content flash before it mounts.
